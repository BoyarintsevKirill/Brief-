# 🚀 AI Marketing Copy Generator - MVP

## Описание решения

**AI Marketing Copy Generator** — это инструмент, который автоматизирует создание вариаций рекламных текстов для разных каналов маркетинга.

### Проблема, которую решает:
Маркетологи в Яндекс Рекламе тратят 2-4 часа в день на написание текстов объявлений для разных каналов (поиск, соцсети, email). Для каждого продукта нужно создавать несколько версий с разными "углами зрения" и адаптировать их под платформу.

### Решение:
За 30 секунд генерируются 3 готовых варианта рекламного текста, адаптированных под конкретный канал и аудиторию, с учётом лучших практик маркетинга.

### Результат для маркетолога:
Вместо написания текста с нуля — проверить 3 варианта и выбрать лучший (или объединить части). Это экономит **60-70% времени** на эту задачу.

## Как это работает (простым языком)

### Этап 1: Ввод информации
Маркетолог рассказывает, какой продукт продвигать:
- 📌 Название продукта
- 📝 Краткое описание (что это такое)
- 👥 Кто это целевая аудитория (руководители, фрилансеры, маркетологи и т.д.)
- 📱 На каком канале разместить (объявление в поиске, пост в инстаграме, email, тиктток)

### Этап 2: Обработка AI
Инструмент отправляет информацию большой языковой модели (Claude) с инструкциями:
- Какой стиль использовать (краткий для поиска, вовлекающий для соцсетей)
- Какую длину текста (80 символов для объявлений, 150-200 для email)
- Что включить (проблема → решение → call-to-action)

### Этап 3: Вывод результата
Маркетолог получает 3 готовых варианта, каждый оптимизирован под канал:
```
ВАРИАНТ 1: Быстрая аналитика для маркетологов
Яндекс Метрика отслеживает действия клиентов на вашем сайте. Видьте, где зависают визиторы, и повышайте конверсию в 2 раза.

ВАРИАНТ 2: Получайте точные данные о клиентах
Настройте отслеживание за 5 минут. Узнайте, откуда приходят покупатели и за что платят больше.

ВАРИАНТ 3: 🎯 Перестаньте гадать - знайте данные
Реальная аналитика вашего сайта. Интеграция за 5 минут. Видно всё. [Подробнее]
```

## Установка и запуск

### Требования
- Python 3.8+
- API ключ Claude от Anthropic (бесплатно на https://console.anthropic.com)

### Шаги установки

1. **Клонируем/скачиваем файлы:**
```bash
cd ~/your-project
# Файл marketing_copy_generator.py должен быть в папке
```

2. **Устанавливаем зависимость:**
```bash
pip install anthropic
```

3. **Устанавливаем API ключ:**
```bash
# На Windows (PowerShell):
$env:ANTHROPIC_API_KEY = "sk-ant-v1-xxxxxxxxxxxx"

# На macOS/Linux:
export ANTHROPIC_API_KEY="sk-ant-v1-xxxxxxxxxxxx"
```

4. **Запускаем программу:**
```bash
python marketing_copy_generator.py
```

## Ограничения и как их преодолеть

### ❌ Проблема: Галлюцинации (AI придумает функции, которых нет)
**Как решить:**
- Всегда проверяйте результат и сравнивайте с реальными возможностями продукта
- Используйте подробное описание в инпуте ("что именно умеет делать продукт")
- Добавьте в промпт примеры хороших текстов для вашего бренда

### ❌ Проблема: API затраты
**Как решить:**
- Каждый запрос стоит ~0.02-0.05 USD
- Для 200 запросов в месяц это ~$4-10
- Устанавливайте лимиты в консоли Anthropic

### ❌ Проблема: Результат не соответствует тону бренда
**Как решить:**
- Добавьте в инструкции примеры удачных текстов
- Укажите стиль (формальный, дружелюбный, технический и т.д.)
- Используйте одного и того же копирайтера как "референса"

### ❌ Проблема: Текст слишком длинный/короткий
**Как решить:**
- Уточните требования по длине в инпуте
- Добавьте жёсткий лимит в промпт ("максимум 80 символов")

## Что сломается и как это поддерживать

### 🔧 Точки отказа

1. **API не доступен или ключ неверный**
   - Решение: проверить доступ в консоли Anthropic, перегенерировать ключ

2. **Timeout (медленный интернет)**
   - Решение: увеличить параметр timeout в клиенте, повторить запрос

3. **Ограничение по rate limit (слишком много запросов)**
   - Решение: добавить задержку между запросами или очередь

4. **AI выдаёт результаты на английском**
   - Решение: добавить в промпт "Результат ВСЕГДА на русском языке"

### 📊 Мониторинг

Добавить логирование:
```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

logger.info(f"Запрос для: {product_name}, токены: {tokens}")
logger.error(f"Ошибка API: {error}")
```

## Что доработать в первую очередь (если проект в прод)

### 🔥 Приоритет 1 (Критично)
1. **Добавить фильтрацию галлюцинаций**
   - Вторая проверка: AI проверяет сгенерированный текст на противоречия с описанием
   - Или использовать RAG (Retrieval-Augmented Generation) с базой функций продукта

2. **Параллелизм**
   - Пользователь может генерировать для нескольких каналов одновременно
   - Использовать async/await или очередь (celery, rq)

3. **Сохранение истории**
   - База данных (PostgreSQL) для сохранения всех генерированных текстов
   - Возможность вернуться и отредактировать нужный вариант

### 🟡 Приоритет 2 (Важно)
4. **Брендинг и стиль**
   - Добавить выбор "бренд гайдлайна" (Яндекс = технический, SberBank = дружелюбный и т.д.)
   - Сохранять примеры успешных текстов и использовать их как референсы

5. **A/B тестирование интеграция**
   - Связать результаты с Яндекс.Директом
   - Показывать какой вариант получил лучший CTR

6. **Веб-интерфейс**
   - Вместо консоли — красивая веб-форма
   - Возможность скопировать текст одним кликом
   - Видеть историю всех генераций

### 🟢 Приоритет 3 (Nice to have)
7. **Мультиязычность**
   - Генерировать тексты для разных стран (Россия, Казахстан, Беларусь)

8. **Автоматизация через n8n**
   - Интегрировать в Slack: напишите команду `/generate-copy` и получите варианты
   - Отправлять результаты в Google Sheets для совместного редактирования

9. **Аналитика использования**
   - Какие каналы генерируются чаще
   - Какие аудитории дают лучшие результаты
   - Стоимость за генерацию
